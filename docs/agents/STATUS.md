# Status (agentic)

**Status** describes how an agentic sits in an engine’s codebase *now*. It is orthogonal to **[Engineship](ENGINESHIP.md)** (Agent vs Resgent).

Canonical roster: [Agentic Registry](AGENTIC_REGISTRY.md) · Full lexicon: [GLOSSARY](../GLOSSARY.md)

## Values

| Status | Meaning |
|--------|---------|
| **Plug-in** | Migrant / bolt-on orientation relative to the mother tree (or archive / dispatch sibling). Can evolve toward **Resident** + **Resgent** Engineship if finetuned for this engine. |
| **Reserve** | Callable specialist — product intent to **spawn / activate on demand** from the mother codebase (lean when called). Runtime spawn API still landing in tokenish. |
| **Resident** | Always-present specialist programmed into the mother engine path. |

## Evolution (product law)

| From → To | When |
|-----------|------|
| Plug-in → Resident (often + Engineship Agent → Resgent) | Code/KG optimized to work natively with this engine |
| Resident → Reserve | Code/KG specialized enough to spawn lean on demand instead of staying always-on |

## Runtime note (tokenish v0.5.x)

| Status | Runtime today |
|--------|----------------|
| Resident | **Yes** — ordinary imports on the hot path |
| Reserve | **Intent labeled** — spawn API not shipped |
| Plug-in | **Taxonomic / migrant labeling** — not an external plugin marketplace |
